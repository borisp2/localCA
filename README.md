# Local CA Deployment Package

A self-contained toolkit for deploying a localhost Certificate Authority on Windows machines. Works online, offline, and in air-gapped environments.

---

## Quick Start

```powershell
# One-liner — installs everything, trusts the CA, opens port 443
powershell -ExecutionPolicy Bypass -File .\Install-LocalCA-Localhost.ps1

# With custom app name and port
.\Install-LocalCA-Localhost.ps1 -AppName "OMNIS" -HttpsPort 5001
```

> **Requires:** Elevated (Run as Administrator) PowerShell prompt.

---

## What's in the Package

| Script | Purpose |
|---|---|
| `Install-LocalCA-Localhost.ps1` | Main installer — provisions OpenSSL, generates CA + server cert, trusts it, configures firewall |
| `Prepare-OfflineBundle.ps1` | Run on an online machine to download prerequisites for offline targets |
| `Uninstall-LocalCA.ps1` | Clean removal of CA, trust entries, firewall rules, and files |
| `Renew-ServerCert.ps1` | Rotate the server certificate without touching the Root CA |

---

## How It Works

```
Install-LocalCA-Localhost.ps1 performs 10 phases:

  Phase 0  Pre-flight        Admin check, port availability
  Phase 1  Prerequisites     Detect or install OpenSSL (9-strategy fallback)
  Phase 2  Directories       Create C:\LocalCA\{private, certs, server, ...}
  Phase 3  Root CA           Generate 4096-bit RSA key + self-signed root cert
  Phase 4  Server Cert       Generate 2048-bit key, CSR, sign with CA (localhost SANs)
  Phase 5  Export Bundles    PFX (for .NET/IIS) + fullchain PEM (for Nginx/Node)
  Phase 6  Trust Store       Import root CA into LocalMachine + CurrentUser stores
  Phase 7  Firewall          Create inbound rule for localhost:port
  Phase 8  Verification      Chain validation + cert detail dump
  Phase 9  Summary           File paths, integration snippets, timing
```

### Certificate Chain

```
[Root CA]  MyApp Localhost Root CA  (4096-bit, 10-year validity)
    │
    └── [Server]  CN=localhost       (2048-bit, 825-day validity)
                  SANs: localhost, %COMPUTERNAME%, 127.0.0.1, ::1
```

### Output Files

```
C:\LocalCA\
├── private\
│   └── ca.key                      ← Root CA private key (ACL-restricted)
├── certs\
│   └── ca.crt                      ← Root CA certificate (distributed to trust stores)
├── server\
│   ├── localhost.key                ← Server private key
│   ├── localhost.crt                ← Server certificate
│   ├── localhost.pfx                ← PFX bundle (for Kestrel / IIS)
│   ├── localhost-fullchain.pem      ← Fullchain PEM (for Nginx / Node)
│   ├── localhost.csr                ← CSR (kept for audit)
│   └── localhost.ext                ← SAN extension file
├── index.txt
├── serial
└── install-ca.log                   ← Detailed installation log
```

## Deployment Scenarios

### Scenario 1: Online Machine (Default)

```powershell
.\Install-LocalCA-Localhost.ps1 -AppName "OMNIS" -HttpsPort 5001
```

OpenSSL is auto-installed via winget → Chocolatey → direct download.

### Scenario 2: Offline / Air-Gapped

**Step 1** — Prepare the bundle on an online machine:

```powershell
.\Prepare-OfflineBundle.ps1 -OutDir ".\bundle" -IncludeInstaller
```

**Step 2** — Copy the `bundle\` folder to the target via USB / network share.

**Step 3** — Run on the target:

```powershell
.\Install-LocalCA-Localhost.ps1 -BundleDir "D:\bundle" -NetworkMode Offline
```

### Scenario 3: Portable OpenSSL (No Install)

If you have a portable `openssl.exe` (e.g. from Git for Windows):

```powershell
.\Install-LocalCA-Localhost.ps1 -OpenSSLExePath "C:\PortableApps\Git\usr\bin\openssl.exe"
```

### Scenario 4: Application Installer Integration

Call from an MSI/EXE installer's custom action:

```
powershell.exe -ExecutionPolicy Bypass -NonInteractive -File "Install-LocalCA-Localhost.ps1" -AppName "OMNIS" -HttpsPort 5001
```

Check `%ERRORLEVEL%`:

| Code | Meaning |
|------|---------|
| 0 | Success |
| 1 | Not elevated (admin required) |
| 2 | OpenSSL not found / not functional |
| 3 | Trust store import failed |
| 99 | Unexpected error |

## Bundle Directory Layout

The `-BundleDir` parameter accepts any combination of these files:

```
bundle\
├── openssl.exe                     ← Portable binary (used directly)
├── openssl\bin\openssl.exe         ← Extracted directory tree
├── openssl.zip                     ← ZIP archive (auto-extracted)
├── openssl-installer.msi           ← MSI (silent install via msiexec)
├── openssl-installer.exe           ← EXE (silent install via /VERYSILENT)
├── MANIFEST.json                   ← Auto-generated by Prepare-OfflineBundle
└── OFFLINE-README.txt              ← Usage instructions for target machine
```

Resolution order: portable → ZIP → MSI → EXE. The first one found wins.

## Certificate Renewal

### Manual Renewal

```powershell
# Renew only if cert expires within 30 days (default)
.\Renew-ServerCert.ps1

# Force renewal + restart a service
.\Renew-ServerCert.ps1 -Force -RestartService "MyAppService"

# Custom threshold
.\Renew-ServerCert.ps1 -RenewThresholdDays 60
```

### Automated Renewal (Task Scheduler)

```powershell
$action  = New-ScheduledTaskAction `
    -Execute "powershell.exe" `
    -Argument "-ExecutionPolicy Bypass -NonInteractive -File C:\LocalCA\Renew-ServerCert.ps1 -RestartService MyAppService"

$trigger = New-ScheduledTaskTrigger -Weekly -DaysOfWeek Monday -At "03:00"

Register-ScheduledTask `
    -TaskName "LocalCA Certificate Renewal" `
    -Action $action `
    -Trigger $trigger `
    -RunLevel Highest `
    -User "SYSTEM"
```

### What Renewal Does

1. Checks current cert expiry against threshold
2. Backs up existing certs to `server\backup-YYYYMMDD-HHmmss\`
3. Generates new key + CSR + cert + PFX + fullchain PEM
4. Keeps the same Root CA (no trust-store changes needed)
5. Optionally restarts a Windows service
6. Prunes backups older than the 5 most recent

## Uninstalling

```powershell
# Remove trust + firewall only (keep files for audit)
.\Uninstall-LocalCA.ps1

# Full removal including all cert files
.\Uninstall-LocalCA.ps1 -RemoveFiles

# Nuclear option — remove everything including OpenSSL
.\Uninstall-LocalCA.ps1 -RemoveFiles -RemoveOpenSSL -Confirm

# Match custom install parameters
.\Uninstall-LocalCA.ps1 -AppName "OMNIS" -HttpsPort 5001 -RemoveFiles
```

The uninstaller:
- Removes Root CA from both LocalMachine and CurrentUser trust stores (by thumbprint or subject fallback)
- Removes the firewall inbound rule
- Optionally deletes all CA files
- Optionally uninstalls OpenSSL (winget → choco → registry MSI lookup)

## Integration Examples

### ASP.NET / Kestrel

In `appsettings.json`:

```json
{
  "Kestrel": {
    "Endpoints": {
      "Https": {
        "Url": "https://localhost:5001",
        "Certificate": {
          "Path": "C:\\LocalCA\\server\\localhost.pfx",
          "Password": ""
        }
      }
    }
  }
}
```

### Nginx

```nginx
server {
    listen 443 ssl;
    server_name localhost;
    ssl_certificate     C:/LocalCA/server/localhost-fullchain.pem;
    ssl_certificate_key C:/LocalCA/server/localhost.key;
}
```

### Node.js

```javascript
const https = require('https');
const fs = require('fs');

const server = https.createServer({
    key:  fs.readFileSync('C:/LocalCA/server/localhost.key'),
    cert: fs.readFileSync('C:/LocalCA/server/localhost-fullchain.pem')
}, app);

server.listen(5001);
```

### curl (Testing)

```bash
curl --cacert "C:\LocalCA\certs\ca.crt" https://localhost:5001
```

Or, since the CA is in the trust store:

```bash
curl https://localhost:5001
```

## Security Notes

- **CA private key** (`private\ca.key`) is ACL-restricted to Administrators and SYSTEM only. Do not relax this.
- **Server cert validity** defaults to 825 days to comply with Apple/Chrome maximum-validity policies.
- **PFX passphrase** is empty by default for automated use. In production, consider setting a passphrase and storing it in a secrets manager.
- The Root CA uses a `C=XX` country code (user-assigned per ISO 3166) to clearly mark it as a local/internal CA. Change `-CaCountry` if needed.
- **Do not use this CA for anything beyond localhost / internal dev.** It is not managed by a proper PKI and has no revocation infrastructure.

## Troubleshooting

| Symptom | Fix |
|---|---|
| "Not elevated" error | Right-click PowerShell → Run as Administrator |
| "OpenSSL not found" in offline mode | Place `openssl-installer.msi` or `openssl.exe` in the bundle directory |
| Browser still shows cert warning | Clear browser cache, restart browser. Check `certmgr.msc` → Trusted Root → look for your CA |
| Chrome NET::ERR_CERT_AUTHORITY_INVALID | Chrome uses the OS trust store. Ensure CA was added to both LocalMachine and CurrentUser |
| Firefox cert warning | Firefox uses its own store. Go to `about:preferences#privacy` → Certificates → Import `ca.crt` |
| Port in use | Check `Get-NetTCPConnection -LocalPort <port>` to find the conflicting process |
| Renewal runs but app still uses old cert | Pass `-RestartService "YourService"` or restart the app manually |
| "Hash mismatch" on MSI | The downloaded MSI doesn't match the pinned hash. Download a fresh copy or update `$OPENSSL_MSI_SHA256` |

## Parameters Reference

### Install-LocalCA-Localhost.ps1

| Parameter | Default | Description |
|---|---|---|
| `-RootDir` | `C:\LocalCA` | Base directory for CA files |
| `-AppName` | `MyApp` | Name in cert subject + firewall rule |
| `-CaValidDays` | `3650` | Root CA lifetime |
| `-ServerValidDays` | `825` | Server cert lifetime |
| `-HttpsPort` | `443` | Port for firewall rule |
| `-BundleDir` | `.\bundle` | Offline package directory |
| `-OpenSSLExePath` | *(empty)* | Hard override for openssl.exe path |
| `-NetworkMode` | `Auto` | `Auto`, `Online`, or `Offline` |
| `-SkipTrustInstall` | `$false` | Skip trust-store import |
| `-SkipFirewallRule` | `$false` | Skip firewall rule creation |
| `-Force` | `$false` | Overwrite existing certs |
| `-LogFile` | `<RootDir>\install-ca.log` | Log file path |

### Renew-ServerCert.ps1

| Parameter | Default | Description |
|---|---|---|
| `-RootDir` | `C:\LocalCA` | CA directory |
| `-AppName` | `MyApp` | Name in cert subject |
| `-ServerValidDays` | `825` | New cert lifetime |
| `-RenewThresholdDays` | `30` | Renew if expiry within N days |
| `-RestartService` | *(empty)* | Service to restart after renewal |
| `-Force` | `$false` | Renew regardless of expiry |

### Uninstall-LocalCA.ps1

| Parameter | Default | Description |
|---|---|---|
| `-RootDir` | `C:\LocalCA` | CA directory to clean up |
| `-AppName` | `MyApp` | For firewall rule matching |
| `-HttpsPort` | `443` | For firewall rule matching |
| `-RemoveFiles` | `$false` | Delete all CA files |
| `-RemoveOpenSSL` | `$false` | Uninstall OpenSSL |
| `-Confirm` | `$false` | Skip confirmation prompt |

### Prepare-OfflineBundle.ps1

| Parameter | Default | Description |
|---|---|---|
| `-OutDir` | `.\bundle` | Bundle output directory |
| `-IncludeInstaller` | `$false` | Copy scripts into bundle |
| `-OpenSSLMsiUrl` | slproweb URL | MSI download URL |
| `-VerifySHA256` | *(empty)* | Expected hash of MSI |
